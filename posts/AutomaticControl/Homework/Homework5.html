3. P5.11

close loop transfer function 是：

T=Ks+KT=Ks+K

因為有考慮到initial condition，再加上這個系統只有一個pole，所以直接把它還原成ODE解：

y′+ky=Aky′+ky=Ak

(y−A)′+k(y−A)=0(y−A)′+k(y−A)=0

因此解為：

y=A+Ce−Kty=A+Ce−Kt

將初始條件帶入，得到：

Q=A+C⇒C=Q−AQ=A+C⇒C=Q−A

因此:

y=A+(Q−A)e−Kty=A+(Q−A)e−Kt

顯然穩態時就是A，所以題目要求的：

∫∞0e2=∫∞0(Q−A)2e−2Kt=(Q−A)22K∫0∞e2=∫0∞(Q−A)2e−2Kt=(Q−A)22K

這就是第一小題問的證明。

接著要最小化K，顯然在K趨近無窮時，誤差平方的積分會趨近零。不過這是不可能的事。現在考慮一個比較實際的狀況：RC電路。假設一開始電容電壓為0(當作Q)，然後接上5V(當作A)(剛好是Arduino的直流工作電壓)，觀察電容的電壓變化：

VC=5−5e−1RCtVC=5−5e−1RCt

考量到電阻用kΩΩ，電容用μFμF，所以RC大約是0.001這個數量級。因此在這個系統K值大約是1000。所以就是：

(0−5)22⋅1000=0.0125(0−5)22⋅1000=0.0125

相較與5V，這個值其實也夠小了的。

接著看迴圈裡要做什麼。首先他要算peak跟peak time。這時候用stepinfo()函數。stepinfo()函數會回傳一個存有step response各種性能指標的結構：

    T_origin = tf([61 * k], [5, 46, 77, 61*k]);
    step(T_origin);
    info = stepinfo(T_origin);

這是還沒做簡化的。接著要把最不重要的pole省去。也就是「實部最負」的pole：

    %找出所有pole
    poles = roots([5  46  77  61*k]) 
    
    %如果有pole的實部比較目前最負得更負，就更新。
    ignore = 1;
    for j = 1:3 
        if real(poles(j)) < real(poles(ignore))
            ignore = j;
        end
    end

    %找出那個pole的值
    compensate = poles(ignore);
    
    %並且把那個pole省略掉
    poles(ignore) = [];

最後使用zpk()函數算出新的transfer function。並且一樣使用stepinfo()函數算出peak跟peak time：

    T_ignored = zpk([], poles, -61*k/compensate/5);
    step(T_ignored);
    info = stepinfo(T_ignored)

另外注意因為DC Gain少除一個minor pole的量值，所以上述程式碼中的transfer function實際上多除了一個compensate，也就是剛剛記下來minor pole的值，來做normalize。

最後，要求K值，使得原本系統的P.O. < 16%。因為已經大致估計出來K = 6時，P.O.大約是6X%左右，而K = 0.7時，P.O.約為1%。因此猜測16%的K值會落在這兩個區間。因為K越大，P.O.越大，大致呈遞增，所以這時候就可以用binary search來求。程式如下：

    iter = 200;
    L = 0;
    R = 6;
    M = 0;
    while iter > 0
        M = (L + R) / 2;
    
        T_origin = tf([61 * M], [5, 46, 77, 61*M]);
        info = stepinfo(T_origin); 
        if(info.Overshoot < 16) L = M;
        else R = M;end
        iter = iter - 1;
    end

Binary Search 原理如下：

  • 每次都把區間(這裡是L=0與R=6之間)平分一半，然後看16是落在左半邊或右半邊。
  • 如果在左半邊，就把右邊界往左設成中點M;
  • 反之，如果是在右邊，就把左邊界往右移動到中點M，並且一直重複。在這裡是重複200次(用iter變數計數)

因為重複200次之後，區間大小變成2−2002−200次方，也就是誤差變成2−2002−200，而且只要200次迭代，可以說是非常經濟。最後只要查看M是多少，即是K值。這裡附上M值及相關的性能指標：
