File Offset

「File Offset」就像是read(), write()距離起始點的位置。

大部分的情況，兩個是一樣的。

剛開檔的時候，通常offset是0。像O_APPEND就不會。

每個「檔案」會用一個struct紀錄，file pointer是其中一個。

stateless vs. statefull

讀寫的時候會改變offset, 也可以用lseek()/fseek()強制移動。

有不能做lseek()的檔案嗎 ? 有！「管線」「FIFO」不行。看範例程式1。

檔案會有「洞」

fork()的兩個程序會用同一個file offset pointer. dup()

dup()會複製一個指向同一個system open file table欄位的不同fd. 可以做到freopen()的效果。
