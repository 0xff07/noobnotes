[SP]第三週筆記

Posted by Fernando Lin on 10, 5, 2016

一個檔案輸入輸出的流程：

  1. User Process 丟一個request給Kernal
  2. Kernel 去 Storage 拿東西到Kernel Buffer（以Block為單位）（跟Cache的原理差不多）
  3. 最後再從Kernel Buffer拿給User Process

這兩個效能瓶頸在兩個地方：

  1. 每次拿一個東西，都會複製兩次
  2. 資料間要保持同步

所以有一個天真的想法就是：把所有東西都放到Kernel 記憶體中，這樣要什麼就有什麼。但是這樣會至少有2個問題：

  1. 多了很多不必要的複製
  2. System Call頻繁，需要花很多時間在mode change上面(從user mode 跟 kernel mode中間)

實驗：call一億次sqrt()與一億次getuid()

而且這樣不是很安全。

計算機組織也碰過這樣的問題，解法是弄一個Cache。這裡也是，多設一個user buffer～

Buffered I/O

這其實跟Cache相關議題的原理一樣。如果東西有在user buffer中，就省去從Kernel搬到User Memory的時間。

3種：

  1. fully buffered：把buffer全塞滿。適合資料小，又需要radom access的時候。不過如果程式突然崩潰，東西就會不見。
  2. line buffered：碰到\n或buffer滿的時候。比如說Terminal
  3. unbuffered：不用buffer, 直接輸出。比如說stderr

buffer可以用setbuf()或setvbuf()設定。要在I/O前就設定好，不然會沒效。
